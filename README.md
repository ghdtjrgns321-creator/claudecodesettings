# claudecodesettings

Claude Code(`~/.claude`)의 전역 설정을 관리하는 저장소.

> **2026-07-16 전면 재설계.** 751세션 실측 데이터로 "문장을 심사하던 층"(매 턴 규율 주입·작문 시험 lint)을 걷어내고, "실체를 계산하는 층"(VERIFY 실행·census 차집합)으로 교체했다. 근거·복원: [docs/2026-07-16-redesign.md](docs/2026-07-16-redesign.md)

## 핵심 원칙

1. **컨텍스트 최소화 — 원칙은 CLAUDE.md 로드, 절차는 필요시 호출**
   - CLAUDE.md에는 판단 규칙만 간결하게(현재 43줄). 매 턴 주입 훅은 두지 않는다.
   - 구체적인 실행 절차는 스킬 형태로 분리, 필요할 때만 로드하여 동작
2. **통제 수준의 분리 — 가이드는 스킬, 제약은 훅**
   - 상황에 따른 판단이 필요한 안내 사항은 스킬로 제공
   - 반드시 지켜야 하는 필수 규칙은 훅으로 강제. 단 훅의 검사 대상은 **산출물·명령의 실체**여야 한다 — 에이전트가 쓴 문장을 심사하는 훅은 작문 시험이 되어 왕복만 낳는다(v1의 교훈)
3. **작업 완료 검증 — 선언이 아니라 계산**
   - 완료 계약(contract)의 검증 항목은 `VERIFY: <실행 명령>` 줄로 박고, Stop 게이트가 직접 실행해 exit 0을 요구
   - 전수 작업은 census 절차(분모 파일 고정 → 항목 흔적 → 차집합 exit 코드)로 증명. 검증 불가 주장은 `미검증` 라벨
4. **설정의 모듈화 — 전역 설정, 프로젝트 설정 분리**
   - 전역 설정(`~/.claude`)에는 상황에 종속되지 않는 범용 환경과 안전장치만 배치
   - 특정 기술 스택 및 도메인 지식은 개별 프로젝트 환경 내에 독립적으로 구성
5. **규칙 추가 전에 데이터**
   - 사고가 나면 규칙부터 만들지 않는다. 트랜스크립트 채굴(교정 분류)·게이트 로그로 원인을 세고, 기계화 가능한 것만 추가
