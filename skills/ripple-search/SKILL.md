# Ripple Search (파급 영향 검색)

## 원칙

다수 파일에 파급되는 변경 후, 구 값(old value)을 전체 검색하여 누락을 방지한다.
"관련 파일 업데이트"는 태스크의 마지막이 아니라 **태스크 체크리스트의 일부**이다.

## 트리거 조건

다음 변경이 발생하면 이 스킬을 적용한다:
- 데이터 스키마 변경 (컬럼 추가/삭제/이름변경)
- 설정값/상수 변경
- 파일/함수/클래스 리네이밍
- 데이터 재생성 (DataSynth, 시드 데이터 등)
- API 엔드포인트 변경
- 환경 변수 키 변경

## 검색 절차

### 1. 전체 검색
변경 완료 후 **구 값(old value)**을 Grep으로 전체 검색한다.
대상 확장자: `md`, `py`, `yaml`, `json`, `toml`, `sql`, `sh`

### 2. 3분류

| 분류 | 처리 |
|------|------|
| **프로젝트 docs** | 즉시 수정 |
| **소스 코드** | 즉시 수정 |
| **테스트 결과 문서** | 재실행 대상으로 분류 (내용 수정 아닌 재실행) |

### 3. 보고
검색 결과를 3분류로 나눠서 사용자에게 보고한다.

## 체크리스트 템플릿

변경 발생 시 태스크 체크리스트에 다음을 포함한다:

- [ ] 구 값 Grep 검색 완료
- [ ] docs/ 내 참조 수정
- [ ] 소스 코드 내 참조 수정
- [ ] 테스트 재실행 및 결과 확인
- [ ] CLAUDE.md / 설정 파일 내 참조 수정
- [ ] test-results/ 문서 재실행 대상 식별

## 주의사항

- 변수명이 다른 형태로 사용될 수 있음 (snake_case, camelCase, UPPER_CASE)
- 문자열 내 참조도 검색 대상 (예: SQL 쿼리, 에러 메시지)
- `.gitignore` 대상 파일도 검색에서 제외하지 않음
