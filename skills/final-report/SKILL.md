# Final Report (전수 기반 최종 보고서)

## 핵심 원칙

**표본이 아니라 전수(全數).** 분모 N은 `scripts/census.py`가 기계로 고정하고, 커버리지 부록이 N/N을 증명해야 완료다. 산출물은 추후 PPT·README(readme-maker)로 가공되는 원료 — 요약하지 말고 세부(수치·설정·코드 인용)를 눌러 담는다.

## 산출물

`<project_root>/FINAL-REPORT/` 디렉토리. 장 구성·장별 필수 요소·시각화(ASCII+Mermaid+배지) 규격은 **references/report-skeleton.md**를 따른다. 문서 스타일(팩트시트 선행·다이어그램 우선·M/N·현재/과거 분리)은 diagram-driven-docs 스킬 계약을 그대로 적용한다.

## 절차

### 1. 센서스 — 분모 N 고정
```bash
python <이 스킬 경로>/scripts/census.py inventory <project_root> --out <scratchpad>/inventory.json
```
**프로젝트 폴더 전체를 순회한다** — 커밋 추적 여부와 무관하게 모든 파일이 대상이다. 설계 문서·데이터 폴더가 커밋에서 제외돼 있어도 빠뜨리지 않는다(핵심 자산이 커밋 밖에 있을 수 있으므로). 하네스·빌드·캐시 디렉토리(`.venv`·`node_modules`·`.git`·`__pycache__` 등)와 스킬 작업공간(`_workspace` — 아웃라인 계약·감사 스크래치)만 기본 제외(목록은 스크립트 상단 `EXCLUDE_DIRS` 상수)하고, 무거운 생성 데이터 폴더를 빼려면 `--exclude DIR`로 명시 제외한다(조용한 누락 금지 — 제외분은 커버리지 장에 기록). `large=true` 파일과 균일 데이터 디렉토리는 "구조 정독"(스키마·대표 표본) 대상으로 표시해 둔다.

### 2. 분산 정독 — 1묶음 1서브에이전트
인벤토리를 디렉토리·도메인 묶음(15~30파일)으로 나눠 팬아웃. 프롬프트는 work-prompt-authoring 스킬로 작성하고, **묶음의 파일 목록을 프롬프트에 명시**해 에이전트가 임의 표본화하지 못하게 한다. 각 에이전트 반환 노트(파일별): 경로 / 역할 1~2줄 / 핵심 수치·상수(출처 라인) / 동작 규칙(이 파일이 정하는 분기·기준·상한·판정 조건 — 값이 아니라 규칙 자체) / **배선 역참조(이 파일을 import·호출하는 곳 — grep 실측, 0곳이면 "미배선" 명기)** / 보고서 인용 후보. 총 30파일 이하 소규모면 팬아웃 없이 직접 정독해도 된다.

**진실 우선순위: 실행 코드 > config·테스트 > docs·주석·docstring·UI 문구.** 파일의 자기서술(현재형 docstring "~한다"·UI 캡션·README 서술)은 동작의 근거가 아니다 — 배선·도달을 확인한 뒤에만 현재형으로 옮긴다. 문서와 코드가 충돌하면 코드를 채택하고 충돌 사실을 노트 비고에 남긴다.

### 3. 팩트시트 — 수치·동작 주장의 단일 출처
노트를 통합해 정본 팩트시트를 scratchpad에 작성. 이후 본문의 모든 수치는 팩트시트에서만 가져온다. 수치만이 아니라 **동작 주장**("X가 Y를 한다"·다이어그램 엣지)도 팩트시트 항목이다 — 출처는 배선 확인된 코드 라인만, 미배선·도달불가는 그 상태 그대로 기재한다. **최신성 앵커**: `git log` 최근 교정 커밋(fix·docs류)을 훑어 "최근에 바로잡힌 사실" 목록을 팩트시트에 박는다 — 낡은 문서 서술이 보고서로 역류하는 것을 차단.

### 4. 집필 — 전체→세세
report-skeleton.md 순서대로: 표지(0) → 개요(1) → **전체 파이프라인(2)** → **부분별 상세(3..K)** → **차별점·특장점** → 검증·ADR → 여정(아키텍처 전환) → **트러블슈팅(운영 사고·디버깅 교훈)** → 커버리지. 여정과 트러블슈팅은 별개 장 — 전환 서사(무엇을 버렸나)와 사고 서사(무엇이 무너졌나)를 분리한다.
표·수치·코드 인용으로 밀도를 채우되, **상세 장(3..K)마다 실증 예시(walked example) ≥1** — 구체 입력 1건이 그 부분을 통과하는 과정을 실제 ID·값과 함께 산문으로 추적한다(파이프라인 장은 end-to-end 1건). 표는 구조를, 실증 예시는 동작감을 준다. PPT·README maker가 곧바로 쓸 데모 서사가 여기서 나온다.

### 5. 검증 — verify exit 0까지
```bash
python <이 스킬 경로>/scripts/census.py verify <project_root> <FINAL-REPORT/마지막 번호의 *_COVERAGE.md>
```
누락 파일이 나오면 그 파일을 정독·반영하고 재실행 — exit 0이 될 때까지 루프.

커버리지 통과는 "다 읽었다"의 증명일 뿐 "맞게 썼다"의 증명이 아니다. 이어서 **주장 감사 패스(필수)**:
집필에 관여하지 않은 에이전트가 각 장의 현재형 동작 서술·다이어그램 엣지·수치 주장을 추출해 코드와
대조하고 판정 표(일치/불일치+근거 라인)를 반환한다 — 수치 스팟체크 3개로 대체 금지. 보고서가 언급한
모듈 전수에 import·호출 grep을 돌려 0곳인 모듈의 현재형 서술은 "미배선"으로 교정한다. 불일치 0이 될
때까지 교정 후 완료를 선언한다.

## Quick Reference

| 단계      | 도구                | 완료 기준                              |
| --------- | ------------------- | -------------------------------------- |
| 센서스    | census.py inventory | N 고정 + inventory.json 저장           |
| 분산 정독 | 서브에이전트 팬아웃 | 모든 묶음 노트 수신 (묶음 M/M)         |
| 팩트시트  | scratchpad 통합     | 본문 수치·동작 주장 출처 100% 팩트시트 |
| 집필      | report-skeleton.md  | 고정 장 + 상세 장 + 커버리지 장        |
| 검증      | census.py verify    | exit 0 (N/N) + 주장 감사 불일치 0      |

## 흔한 실패

- **표본 몇 개 읽고 "전수" 주장** → verify exit 0 없이는 완료 선언 금지 (hollow-PASS).
- **분모에서 조용히 제외** → `--exclude` 사용 시 커버리지 장에 제외 표(디렉토리·건수·사유) 필수, verify도 동일 `--exclude`로 실행.
- **요약 기억으로 수치 기입** → 팩트시트 먼저, 본문은 팩트시트만 인용.
- **수치·상수만 뽑고 그 값을 구동하는 규칙을 흘림** → 존재는 기록됐으나 동작이 빠짐(존재≠동작). 규칙이 산문·설정에 있는 파일(프롬프트·검증기·라우터·플래그)이 특히 취약 — 값이 아니라 규칙 자체를 전사한다.
- **파일 자기서술 전사** → 현재형 docstring("~감사한다")·UI 캡션·낡은 README가 실행 코드를 이김. 2026-07-16 실측: 생성 보고서의 치명 불일치 14건 전부 이 계열(모델 환각 0건) — 경로·수치는 census·팩트시트가 지켰지만 동작 서술은 게이트 밖이었다. 현재형 동작 서술은 배선 grep 후에만.
- **구현 존재를 동작으로 승격** → 존재≠배선≠도달. import 역참조 0곳이면 미배선이고, 분기 코드는 라이브 호출 경로가 실제로 도달하는지까지 확인해야 "동작한다"로 쓸 수 있다.
- **기존 FINAL-REPORT 갱신 요청** → 기존 장 구성을 존중하되 센서스부터 다시(N 변동 반영).
- **대형 파일 전문 정독 시도** → `large=true`는 구조 정독(스키마+대표 표본)으로 대체하고 커버리지 비고에 표기.
- **표·수치만 있고 실증 예시 0개** → 밀도는 높으나 "실제로 어떻게 흐르는지"가 안 보인다. 상세 장(3..K)·파이프라인 장마다 walked example ≥1(구체 입력 1건을 종착까지 추적)을 넣어라 — 다운스트림(PPT·README)이 스스로 예시를 지어내지 않게.
