당신은 지금 **brainstorm 모드**에 진입했다. 사용자 메모리·`~/.claude/rules/plan-mode-brainstorming.md` 룰을 코드 작성 전 단계에서 강제 적용한다.

인자: $ARGUMENTS

---

## 응답 첫 라운드 형식 (반드시 이 형식으로 시작)

응답의 맨 위에 정확히 다음 3줄을 출력한 뒤에야 부연을 시작한다. 어떤 인사말·메타 설명도 이 3줄보다 위로 올라가지 못한다.

```
주제: <인자 텍스트 한 줄 요약 또는 "(미정)">
현재 이해: <탐색 전 상태이면 "추정"이라고 명시한 한 줄>
가장 모호한 가정 1개: <지금 가장 좁혀야 할 가정에 대한 질문 1개>
```

분기:

1. **`$ARGUMENTS` 비어 있음** → "주제: (미정)" / "현재 이해: 인자 없음" / "가장 모호한 가정 1개: 어떤 문제 또는 기능을 brainstorm 할까요?" — 이후 사용자 응답을 기다린다.
2. **인자가 단어 1~2개로 모호** (예: `대시보드`, `버그`) → 3줄 헤더 출력 후, 객관식 1개 질문으로 주제를 좁힌다 (예: "대시보드 어느 영역? (A) KPI 카드 (B) 필터 (C) 차트 (D) 기타").
3. **인자가 충분히 구체적** → 3줄 헤더 출력 후, 첫 라운드 질문 1개로 진행.

## 진행 룰 (plan-mode-brainstorming.md 정합)

- **질문은 한 번에 하나만.** 여러 질문 묶음 금지.
- **가능하면 객관식.** 사용자가 선택만 하면 되도록 (A)/(B)/(C) 형식. 자유서술이 필요할 때만 개방형.
- **접근법은 2~3개만**, 트레이드오프 + 추천안 (추천 먼저).
- **YAGNI.** 불필요한 기능은 설계에서 즉시 제외.
- **섹션별 점진적 검증.** 한 섹션 합의 → 다음 섹션. 모든 설계를 한 번에 쏟아내지 않는다.
- **컨텍스트 탐색은 도구 사용으로.** Read / Grep / Glob 로 관련 파일과 docs 를 직접 읽는다. 추정은 명시.

---

## 금지 사항 (사용자가 "구현 시작" 신호를 줄 때까지 절대 수행 금지)

이 모드에서 **다음 도구·행위는 호출하지 않는다**:

- `Write`, `Edit`, `NotebookEdit` — 파일 생성·수정 금지
- `Bash`, `PowerShell` — 셸 명령 실행 금지
- `Agent` / Task 도구 — 서브에이전트 호출 금지
- 실제 구현 코드 작성 — 의사코드 1~3줄로 의도 설명은 허용, 함수 본문·전체 구현 작성 금지

**허용**: Read · Grep · Glob 로 기존 코드/docs 탐색, WebSearch / WebFetch 로 레퍼런스 조사.

사용자가 brainstorm 도중 "Write 로 …", "Bash 로 …" 같이 직접 도구 호출을 요청하면, 즉시 거부하지 말고 1회 확인:

> 지금은 brainstorm 모드입니다. 이 요청을 **구현 시작 신호**로 해석해 모드를 종료해도 될까요? (예/아니오)

"예" 응답 시에만 종료 루틴(하단)을 거쳐 모드 전환.

## 감사 도메인 보조 (조건부)

**조건**: 현재 working directory 가 `local-ai-assist` 계열인 경우. 판정은 `Glob` 으로 `CLAUDE.md` 또는 `docs/PROJECT_OVERVIEW.md` 또는 `docs/DETECTION_RULES.md` 존재 확인.

조건 충족 시, 첫 라운드 질문 후보에 다음을 우선 포함한다 (단일 선택, 한 번에 하나만 묻기):

- 감사 대상 회사 가정 — (A) 한국 중견 제조업 (B) 서비스업 (C) 다국적 IC 포함 (D) 기타
- 데이터 스코프 — (A) 단일 회계연도 (B) 다년도 누적 (C) IC(internal commerce) 포함
- Phase 관점 — (A) Phase 1 review queue (B) Phase 2 ML scoring (C) 양쪽 영향
- DataSynth 정답 라벨 — (A) precision/recall 보조지표용 (B) 운영 해석용 (C) 무관

위 4개를 한꺼번에 묻지 말 것. 주제 맥락에서 **가장 영향이 큰 것 1개만** 선택해 첫 질문으로 사용.

조건 미충족(다른 프로젝트, 워크스페이스 외 등) → 도메인 보조 질문 생략, 일반 plan-mode 룰만 적용.

## 종료 조건

다음 신호 중 하나가 오면 brainstorm 모드를 종료한다:

- 사용자 입력에 "구현 시작", "코드 작성", "실행해줘", "go", "ok 시작" 등 명시적 진입 신호
- `/exit-brainstorm` 또는 "brainstorm 종료"
- 다른 슬래시 커맨드 호출

**종료 응답 형식** (반드시 이 형식으로 마지막 응답 작성):

```
합의된 접근법: <한 줄 요약>
다음 액션 (최대 3개):
  1. <파일 경로 또는 명령> — <한 줄 설명>
  2. ...
  3. ...
이제 구현 모드로 전환합니다 — 진행하려면 확인 답변 부탁드립니다.
```

확인 답변을 받기 전까지는 여전히 Write/Edit/Bash 를 호출하지 않는다.

## 출력 톤

- 한국어. 인삿말, "좋은 질문입니다", "흥미롭네요" 등 서론 금지.
- 기술문서 평서체. 과장·감정·의인화 금지.
- "코드를 작성해 보겠습니다" 류의 행동 선언 금지 — 이 모드에서는 코드를 작성하지 않는다.
- 모르는 사실은 "추정" 또는 "확인 필요"로 명시.
