# Plan 모드 Brainstorming 규칙

Plan 모드에서 설계를 수립할 때 아래 원칙을 따른다.

## 핵심 원칙

- **질문은 한 번에 하나씩** — 여러 질문으로 압도하지 않기
- **객관식 선호** — 가능하면 선택지 제공
- **2~3개 접근법 제안** — 트레이드오프와 추천안을 함께 제시, 추천안 먼저
- **YAGNI** — 불필요한 기능은 설계에서 제거
- **점진적 검증** — 섹션별 설계 제시 → 승인 → 다음 단계

## 프로세스

1. **컨텍스트 탐색** — 관련 파일, docs, 최근 커밋 확인
2. **명확화 질문** — 목적, 제약, 성공 기준 파악 (한 번에 하나)
3. **접근법 제안** — 2~3개 방안 + 트레이드오프 + 추천
4. **설계 제시** — 복잡도에 비례한 상세도, 섹션마다 확인
5. **최종 계획 작성** — plan 파일에 확정된 설계 반영

## 설계 품질 기준

- 시스템을 작은 단위로 분해: 하나의 목적, 잘 정의된 인터페이스, 독립적 테스트 가능
- 기존 코드베이스 패턴을 먼저 탐색하고 따르기
- 관련 없는 리팩토링은 포함하지 않음
- "간단한 것"이라도 설계 단계를 건너뛰지 않음 (짧게는 가능)

## 범위 판단

- 여러 독립 서브시스템이 포함된 요청이면 즉시 분해 제안
- 단일 구현 계획으로 충분한 범위인지 점검
- 너무 크면 서브 프로젝트로 나누고 순차 진행
