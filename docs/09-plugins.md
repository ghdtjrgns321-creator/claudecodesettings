# 09. Plugins — 플러그인

> 플러그인 = **스킬·에이전트·훅·LSP를 묶은 외부 배포판.**

## 스킬·에이전트·훅과 차이

|                        | 만드는 주체 | 저장 장소                    | repo 포함 | 유지보수                            |
| ---------------------- | ----------- | ---------------------------- | --------- | ----------------------------------- |
| 스킬/에이전트/훅(고유) | 내가 작성   | `skills/`·`agents/`·`hooks/` | ✅        | 내가                                |
| **플러그인**           | **제3자**   | `plugins/cache/`             | ❌(캐시)  | **제작자가 원격으로 자동 업데이트** |

## 설치된 플러그인

마켓플레이스: `claude-plugins-official`(Anthropic 공식) · `harness-marketplace`(revfactory) · `im-not-ai`(epoko77-ai) · `context-mode`(추가만).

| 플러그인              | 종류                 | 역할                                                                       | 상태                                  |
| --------------------- | -------------------- | -------------------------------------------------------------------------- | ------------------------------------- |
| **harness**           | 팩토리               | 도메인 설명 → 에이전트 팀+스킬을 프로젝트 `.claude/`에 스캐폴딩(6 팀 패턴) | 활성                                  |
| **humanize-korean**   | 스킬 3 + 에이전트 10 | 한글 AI 티 탐지·윤문 (Fast + strict 5인 파이프라인)                        | 활성 (`settings.local.json`에서 배선) |
| **pyright-lsp**       | LSP                  | Python 코드 인텔리전스(타입체크·정의 이동·심볼)                            | 활성                                  |
| **rust-analyzer-lsp** | LSP                  | Rust 코드 인텔리전스                                                       | 활성                                  |
| context-mode          | (마켓)               | 세션 컨텍스트 연장                                                         | 마켓만 추가, 미설치                   |

> LSP 플러그인은 Claude가 그 언어의 코드를 더 정확히 이해하게 돕는 언어 서버 연동(스킬 아님).

## harness — 팀 아키텍처 팩토리

revfactory(robin) 제작. **도메인 한 문장 → 전문 에이전트 팀 + 그들이 쓸 스킬을 프로젝트 `.claude/`에 생성.**

| 패턴        | 구조                      | 적합                        |
| ----------- | ------------------------- | --------------------------- |
| 파이프라인  | 순차 의존 A→B→C           | 정해진 공정(원고→편집→발행) |
| 팬아웃/팬인 | 병렬 독립 후 취합         | 다각도 조사·다중 소스       |
| 전문가 풀   | 상황별 선택 호출          | 도메인별 질의 분배          |
| 생성-검증   | 생산자+검토자 품질 게이트 | 품질 보증 필요한 산출       |
| 감독자      | 중앙 에이전트가 동적 분배 | 작업량 예측 불가한 조율     |
| 계층적 위임 | 상위→하위 재귀적 위임     | 대규모 다단계 프로젝트      |

## humanize-korean

epoko77-ai 제작. 한글 텍스트의 AI 티(번역투·기계적 병렬·피동태 남용 등 10대 카테고리 40+ 패턴)를 탐지·분류해 내용은 그대로 두고 문체만 재작성.

| 진입             | 용도                                         |
| ---------------- | -------------------------------------------- |
| `/humanize`      | Fast 모드 윤문 (`--strict`면 5인 파이프라인) |
| `/humanize-redo` | 최근 윤문 결과 2차 조정                      |
| 자동 트리거      | "AI 티 없애줘", "번역투 제거" 등             |
