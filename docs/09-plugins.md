# 09. Plugins — 플러그인

> 플러그인 = **스킬·에이전트·훅·LSP를 묶은 외부 배포판.** 

## 스킬·에이전트·훅과 뭐가 다른가

|                        | 만드는 주체 | 저장 장소                    | repo 포함 | 유지보수                            |
| ---------------------- | ----------- | ---------------------------- | --------- | ----------------------------------- |
| 스킬/에이전트/훅(고유) | 내가 작성   | `skills/`·`agents/`·`hooks/` | ✅        | 내가                                |
| **플러그인**           | **제3자**   | `plugins/cache/`             | ❌(캐시)  | **제작자가 원격으로 자동 업데이트** |


## 설치된 플러그인

출처 마켓플레이스: `claude-plugins-official`(Anthropic 공식) + `harness-marketplace`(revfactory) + `context-mode`(추가만).

| 플러그인              | 종류      | 역할                                                                       | 상태                |
| --------------------- | --------- | -------------------------------------------------------------------------- | ------------------- |
| **superpowers**       | 스킬 14개 | 작업 방식 하네스(TDD·디버깅·검증·계획·리뷰…)                               | 설치·활성           |
| **harness**           | 팩토리    | 도메인 설명 → 에이전트 팀+스킬을 프로젝트 `.claude/`에 스캐폴딩(6 팀 패턴) | 설치·활성           |
| **pyright-lsp**       | LSP       | Python 코드 인텔리전스(타입체크·정의 이동·심볼)                            | 설치·활성           |
| **rust-analyzer-lsp** | LSP       | Rust 코드 인텔리전스                                                       | 설치·활성           |
| context-mode          | (마켓)    | 세션 컨텍스트 연장                                                         | 마켓만 추가, 미설치 |

> LSP 플러그인은 Claude가 그 언어의 코드를 더 정확히 이해하게 돕는 언어 서버 연동(스킬 아님).

## harness — 팀 아키텍처 팩토리

revfactory(robin) 제작. **도메인 한 문장 → 전문 에이전트 팀 + 그들이 쓸 스킬을 프로젝트 `.claude/`에 생성.**

도메인에 맞는 **6가지 팀 아키텍처 패턴** 중 하나로 팀을 구성한다.

| 패턴        | 구조                      | 적합                        |
| ----------- | ------------------------- | --------------------------- |
| 파이프라인  | 순차 의존 A→B→C           | 정해진 공정(원고→편집→발행) |
| 팬아웃/팬인 | 병렬 독립 후 취합         | 다각도 조사·다중 소스       |
| 전문가 풀   | 상황별 선택 호출          | 도메인별 질의 분배          |
| 생성-검증   | 생산자+검토자 품질 게이트 | 품질 보증 필요한 산출       |
| 감독자      | 중앙 에이전트가 동적 분배 | 작업량 예측 불가한 조율     |
| 계층적 위임 | 상위→하위 재귀적 위임     | 대규모 다단계 프로젝트      |
