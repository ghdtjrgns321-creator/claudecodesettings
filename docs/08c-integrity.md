# 08c. 무결성 하네스 (파일 저장 시 자동 검사)

> 파일을 쓸 때마다(PostToolUse:Write\|Edit) 인코딩·하드코딩·서식을 자동 검사·정리한다.
> ← [08. 하네스 개요](08-harness.md)

3개 훅이 저장 직후 순서대로 돈다.

## ① post_write_check.py — 한글 인코딩 가드 (하드 차단) + ruff

| 검사                                                           | 동작                                      |
| -------------------------------------------------------------- | ----------------------------------------- |
| **UTF-8 디코딩 실패**                                          | `exit 2` 하드 차단                        |
| **U+FFFD(대체문자) ≥ 1**                                       | `exit 2` 차단 — 인코딩 손상의 결정적 증거 |
| **cp949→utf-8 깨짐 시그니처** (알려진 깨짐 바이트열 ≥2개 검출) | `exit 2` 차단                             |
| **한글 자모 단독 비율 > 30%** (정상 한글은 >95%가 완성형 음절) | `exit 2` 차단                             |
| **.py 파일**                                                   | ruff format + check(--fix) 자동 실행      |

→ **한글 파일이 PowerShell 라운드트립·Set-Content 등으로 깨지는 사고를 원천 차단.** 이 저장소를 만드는 내내 한글 문서가 안 깨진 건 이 훅 덕분(실제로 여러 번 발동).

## ② literal_lint.py — 하드코딩 경고 (차단 안 함)

소스코드에 **계산을 구동하는 리터럴**이 박히면 경고:

| 종류      | 패턴               | 의미                   |                         |
| --------- | ------------------ | ---------------------- | ----------------------- |
| 연도      | `\b(19             | 20)\d{2}\b`            | 분석 연도 하드코딩 의심 |
| 긴 식별자 | `\b\d{6,}\b`       | corp_code 등 ID 리터럴 |                         |
| 거대 상수 | `1e8` 등 지수 표기 | 절대 임계 하드코딩     |                         |

- 대상: 소스 확장자(.py·.js·.ts·.go 등). **제외**: `/config`·`/tests`·`/.claude`·주석.
- 출력: "이 값이 계산을 분기·임계·대상선정에 쓰면 데이터·인자·config로 빼라. 단순 상수면 무시." → **경고만, 차단 안 함.**
- 끄기: 프로젝트에 `.claude/state/.nolint` 두면 비활성(거대숫자가 정상인 프로젝트).

## ③ md_table_format.py — 마크다운 표 자동 정렬

- GFM 표의 `|` 컬럼 폭을 자동 정렬. **전각(한글·CJK·이모지)은 2칸**으로 계산해 소스에서도 표가 반듯하게 보인다.
- 코드펜스(```) 내부 표는 안 건드림. 셀 부족 행은 빈칸 패딩(데이터 손실 0).
- **utf-8 strict로만 read/write**, 디코딩 실패 시 파일을 건드리지 않음(한글 손상 방지).

## 왜 훅인가

- 인코딩 손상은 한 번이면 파일 전체가 깨진다 → **하드 차단**이 맞다(①).
- 포맷·린트는 "매번 신경 쓰기"가 비현실적 → 저장 시 **자동**이 맞다(②③).
