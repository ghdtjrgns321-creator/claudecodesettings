## 새 구조의 원칙

1. **판정자는 산출물을 읽는다. 에이전트의 서술은 읽지 않는다.** 검증 = 실행 가능한 명령(VERIFY:·census_diff)의 exit 코드.
2. **검증 불가 주장은 세탁하지 않고 `미검증` 라벨로 남긴다.**
3. **모델 통제는 규칙이 아니라 분업으로.** 판단·스펙은 상위 모델 짧게, 실행·검증은 좁은 스펙의 샤드로. 교정 2회에도 방향이 안 잡히면 세션을 버리고 확정사항 5줄로 재시작 (오염 컨텍스트는 회복 안 됨).
4. **규칙 추가 전에 데이터.** 사고가 나면 규칙부터 만들지 말고, 이 문서 방식(교정 채굴·게이트 로그)으로 원인을 세고 나서 기계화 가능한 것만 추가한다.
