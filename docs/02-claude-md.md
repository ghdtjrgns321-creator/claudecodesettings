# 02. CLAUDE.md 구성 방식

전역 규칙 파일. **84줄 / ~2.1k 토큰** (권장 5k 이하). 매 세션 로딩되므로 원칙만 담고 상세는 링크로 위임한다.

## 뿌리 = 카파티(Andrej Karpathy) 4원칙

§0 Core 4줄은 Karpathy의 "LLM 코딩 4원칙"의 한글 요약이다. LLM 기본 실패 4종을 막는 고삐:

| #   | 원칙                                      | 막는 실패              |
| --- | ----------------------------------------- | ---------------------- |
| 1   | 모르면 멈추고 묻는다 (가정·혼란 드러내기) | 가정하고 그냥 달림     |
| 2   | 요청한 것만, 최소로                       | 100줄이면 될 걸 부풀림 |
| 3   | 수정 전 읽고, 외과적으로                  | 안 시킨 것까지 건드림  |
| 4   | '됐다'는 증거로 (성공기준 세우고 검증)    | 검증 없이 "됐다"       |

## 슬림 원칙 (왜 84줄인가)

- **방법론은 스킬로 위임.** §9·§10의 상세 검증 절차는 superpowers 스킬 + completion_gate 훅이 강제하므로 CLAUDE.md엔 원칙 2줄만.
- **강조어 절제.** `YOU MUST`/`IMPORTANT`는 준수율을 올리지만 남발하면 희석 → 진짜 non-negotiable 2곳(AI 커밋 문구 금지·한글 인코딩)에만.
- **도메인 예시 금지.** 회계·감사 같은 예시는 전역에 안 두고 프로젝트 CLAUDE.md로.
- **목차 내장.** 상단에 §0~§10 한 줄 목차 → 놓침 방지.

## 관리 규칙

- 규칙을 추가하기 전에 자문: "이건 훅으로 강제할 수 있나(→훅), 가끔 쓰는 절차인가(→스킬), 매 턴 참인 원칙인가(→CLAUDE.md)?"
- 슬림 유지: 새 규칙 넣을 때 대응하는 옛 규칙/중복을 함께 정리.
- 백업: 큰 변경 전 `CLAUDE.md.pre-*.bak` + git.
