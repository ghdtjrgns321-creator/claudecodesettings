# 06. Commands — 슬래시 커맨드

커맨드 = `/이름` 으로 **직접 호출**하는 프롬프트 템플릿(`commands/*.md`). 스킬(자동)과 달리 수동 실행.

## 커맨드 3개

| 커맨드             | 기능                                                  | superpowers 중복                   |
| ------------------ | ----------------------------------------------------- | ---------------------------------- |
| `/brainstorm`      | 코드 전 가정 좁히고 접근법 2~3개 비교(Plan 모드 진입) | superpowers:brainstorming (자동판) |
| `/dev-docs`        | 전략 계획 + 작업 분해 문서 생성                       | superpowers:writing-plans          |
| `/dev-docs-update` | 대화 요약 직전 개발 문서 갱신                         | 없음(고유)                         |

## 유지 판단

- 3개 다 유지. 슬래시 커맨드는 **직접 호출**이라 자동 스킬(superpowers)과 공존해도 충돌·소음 없음.
- `/dev-docs-update`는 대체 불가한 고유 기능(컨텍스트 압축 대비).
- `/brainstorm`·`/dev-docs`는 superpowers 스킬과 겹치지만, "명시적으로 부르고 싶을 때" 남겨둠.

## 커맨드 vs 스킬 vs 에이전트

|          | 호출                  | 용도                    |
| -------- | --------------------- | ----------------------- |
| 커맨드   | `/이름` 수동          | 정해진 프롬프트 실행    |
| 스킬     | description 자동 매칭 | 상황별 방법론·가이드    |
| 에이전트 | Task 파견             | 독립 컨텍스트 자율 작업 |

## 운영 규칙

- 커맨드 추가 시 superpowers·기존 커맨드와 중복 확인. 고유 가치가 있을 때만.
