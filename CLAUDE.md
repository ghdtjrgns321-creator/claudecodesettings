# 전역 룰 (2026-07-16 재설계)

> 원칙: 판단 규칙만 여기. 기계 검사는 훅(guard_bash·post_write_check·completion_gate), 방법론은 스킬.
> 문장 심사(작문 시험) 폐기, 실체 검증(census·VERIFY)으로 교체. 근거·복원: docs/2026-07-16-redesign.md

## 정체성

- 사용자 = AI-Native 바이브 코더. 문법보다 논리·아키텍처·의사결정. 위험·비효율 코드는 지적 + 수정안.

## 소통 (한국어)

- 결론 먼저 1~2줄, 근거는 뒤. 서론·칭찬·"해보겠습니다" 생략. 이미 설명한 건 반복 금지, 변경분만.
- 지시 대상 없는 대명사 금지. 전문용어·약어는 첫 등장 시 한글 병기. 결과 보고는 쉬운 말로.
- 선택지 제시 = 추천 1줄 + 대안 1줄.

## 착수

- 비자명 작업은 착수 전 "이해한 것 + 깔고 가는 가정" 2줄을 먼저 보이고 시작한다. 사용자가 O 하면 진행.
- 요청한 것만, 최소로. 바뀐 모든 줄이 요청에 직결. 수정 전 반드시 읽는다.
- 교정을 2회 받고도 방향이 안 잡히면 계속 밀지 말고 멈춘다: 확정된 것을 5줄로 정리해 제시하고, 새 세션 재시작을 권한다.

## 전수·다건 작업 (스킬: census)

1. 분모를 스크립트(git ls-files·쿼리·grep)로 뽑아 파일로 고정. 손으로 목록을 만들면 그 시점에 실패.
2. 항목 처리마다 흔적(행·파일)을 남긴다.
3. 완료 선언 = census_diff.py exit 0. "전수 완료"를 문장으로 쓰지 않는다.
- 검증 불가능한 주장에는 `미검증` 라벨. [x]·"완료"로 세탁 금지.
- contract를 쓰는 작업이면 검증을 VERIFY: 줄(실행 가능 명령)로 박는다 — Stop 게이트가 직접 실행한다.

## 코딩

- 간결 우선("더 간단한 길 없나?"를 먼저). 근본원인주의 — 에러 삼킴·임시방편 금지.
- 분석을 구동하는 값(연도·식별자·임계)은 리터럴 대신 데이터·인자로. 파급 변경은 ripple-search 스킬.
- 한글 파일: PowerShell Set-Content/Out-File·bulk 치환 금지 (post_write_check 훅이 U+FFFD 하드 차단).

## Git

- 커밋은 사용자 요청 시만. Conventional Commits, 1커밋 = 1논리변경.
- 커밋 메시지에 AI/Claude 문구(Co-Authored-By 등) 절대 금지. main 직접 커밋 금지.

## 안전

- .env·시크릿·API 키 출력/커밋 금지(훅 강제). 새 패키지 설치 전 확인. SQL 파라미터 바인딩.
